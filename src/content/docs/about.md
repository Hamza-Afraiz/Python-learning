## What this site is

**Backend Engineering, Explained** is a free, open educational resource that
teaches the **reasoning** behind real backend systems — not just commands, but
the *why* behind every architectural decision.

It is organized into two parts:

- **Visual Guides** — diagram-first explainers of the big topics: system
  architecture, data & storage, messaging & events, async Python, and
  observability.
- **Q&A Knowledge Base** — a growing collection of **100+ real questions** about
  backend engineering, each answered in plain English.

## Topics covered

- **Async Python** — the event loop, blocking vs non-blocking, the GIL,
  Uvicorn & Gunicorn
- **Databases** — PostgreSQL, MongoDB, Cassandra, transactions, MVCC, WAL,
  replication, pgvector
- **Messaging & events** — Kafka, RabbitMQ, Celery, Temporal, the outbox pattern
- **APIs** — FastAPI, dependency injection, auth, role-based access control
- **Infrastructure** — Docker, Docker Compose, deployment patterns
- **Observability** — Prometheus, Grafana, OpenTelemetry, Jaeger, Loki
- **Distributed systems** — sagas, idempotency, eventual consistency, failure
  modes

## Who it's for

Backend and full-stack engineers preparing for **system-design interviews**, or
anyone who wants the *reasoning* behind real infrastructure choices.

## How it's built

This site is fully **static** — built with [Astro](https://astro.build) and
[Starlight](https://starlight.astro.build), hosted free on GitHub Pages.
Diagrams are rendered with [Mermaid](https://mermaid.js.org/). It's fast,
searchable, works on mobile, and the source is open.
